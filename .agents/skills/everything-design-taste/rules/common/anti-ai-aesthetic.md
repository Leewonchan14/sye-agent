# Anti-AI Aesthetic Rules

## Visual
1. Never default to Inter, Roboto, or Arial without explicit justification.
2. Never use purple/blue gradient backgrounds as a default choice.
3. Never create 3-column feature card grids with centered icons as the primary layout pattern.
4. Never use generic hero sections with centered headline + subtext + single CTA button unless the content genuinely calls for it.
5. Never use border-radius: 12px + box-shadow as the default card treatment. Consider alternatives.
6. Never use generic stock illustration styles (Humaaans, unDraw defaults) without customization.
7. Vary aesthetic choices between projects. No two designs should converge on the same visual solution.

## Language
1. Never open with "In today's..." or "Imagine a world where..."
2. Never use "leverage," "utilize," "synergy," "robust," "seamless," "cutting-edge," or "best-in-class."
3. Never use filler transitions: "Furthermore," "Moreover," "Additionally," "That being said."
4. Limit em-dashes to one per paragraph maximum. Prefer commas or periods.
5. Never use generic CTAs: replace "Get Started" with what users are actually starting, "Learn More" with what they will learn.
6. Never start a paragraph with "It's important to note that" or "It goes without saying."
