# Intentionality Rules

1. **Every element earns its place.** Before adding any element, ask: "What problem does this solve for the user?" If the answer is "it's expected" or "other sites have it," reconsider.

2. **No cargo cult design.** Don't add hamburger menus because apps have them. Don't add hero sections because landing pages have them. Use patterns when they serve the user, not when they serve convention.

3. **Design decisions are strategic decisions.** Color, typography, spacing, and layout are tools for directing attention and building trust. Treat them with the same rigor as product decisions.

4. **Delete before you add.** When something isn't working, try removing elements before adding new ones. Simplification solves more problems than addition.

5. **States are features.** Empty states, error states, loading states, and edge cases are not afterthoughts. They are part of the user experience and must be designed intentionally.

6. **Whitespace is active.** Space between elements communicates relationships. More space = less related. Less space = more related. Space is not "empty," it's information.
