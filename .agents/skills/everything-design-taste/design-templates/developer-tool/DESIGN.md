# Developer Tool Design System

## Color Tokens
```css
:root {
  --color-bg-primary: #0D1117;
  --color-bg-secondary: #161B22;
  --color-bg-tertiary: #21262D;
  --color-text-primary: #E6EDF3;
  --color-text-secondary: #8B949E;
  --color-accent: #58A6FF;
  --color-accent-hover: #79C0FF;
  --color-border: #30363D;
  --color-success: #3FB950;
  --color-warning: #D29922;
  --color-error: #F85149;
  --color-info: #58A6FF;
}
```

## Typography
- Display: "Geist Sans", sans-serif (500, 700)
- Body: "Geist Sans", sans-serif (400, 500)
- Mono: "Geist Mono", monospace
- Code blocks use mono. Everything else uses sans.

## Spacing
4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 (tight scale, dense layouts)

## Radius
Small: 4px | Medium: 6px | Large: 8px (subtle, not rounded)

## Shadows
Minimal. Use borders and background differentiation.

## Motion
- Micro: 100ms ease-out
- Standard: 200ms ease-out
- Fast and functional. Never decorative.

## Personality
Built for people who notice when something is 1px off. Information density is a feature, not a problem. Keyboard-first. Dark by default. Monospace where precision matters. The aesthetic of a well-configured terminal, extended to a graphical interface. Respect the user's intelligence.
